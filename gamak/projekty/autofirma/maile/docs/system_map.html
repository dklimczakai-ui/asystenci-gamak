📍 SZYBKI START — co możesz zrobić TERAZ

Curl działa TYLKO w terminalu. NIE w przeglądarce, NIE w Notion, NIE w Gmail. Otwórz PowerShell (Start → wpisz "powershell" → Enter) ALBO Git Bash (jeśli masz Git zainstalowany — Start → "git bash"). Potem klikasz "kopiuj" pod komendą poniżej i wklejasz w terminal.

A. Pokaż drafty czekające na moją decyzję

curl https://jb69vusexb.execute-api.eu-central-1.amazonaws.com/agent/inbox

Zwraca JSON z listą draftów ze status=PENDING (drafty wygenerowane przez AI, czekają aż zdecydujesz: send/reject/amend).

B. Wyślij draft (po review treści)

curl -X POST https://jb69vusexb.execute-api.eu-central-1.amazonaws.com/agent/action -H "Content-Type: application/json" -d '{"action":"send","draft_id":"WSTAW_TUTAJ_DRAFT_ID","dry_run":true}'
Najpierw "dry_run":true — pokaże co BYŁOBY wysłane, bez wysyłania. Jak treść OK, zmień "dry_run":true na "dry_run":false (lub usuń) i pchnij realny send.

C. Odrzuć draft (zły, nie podoba mi się)

curl -X POST https://jb69vusexb.execute-api.eu-central-1.amazonaws.com/agent/action -H "Content-Type: application/json" -d '{"action":"reject","draft_id":"WSTAW_DRAFT_ID"}'

D. Popraw draft z hintem (Drafter zrobi nowy)

curl -X POST https://jb69vusexb.execute-api.eu-central-1.amazonaws.com/agent/action -H "Content-Type: application/json" -d '{"action":"amend","draft_id":"WSTAW_DRAFT_ID","hint":"Pisz krócej, dodaj że to jest pilne"}'

E. Zapisz task / decyzję / fakt do S3 (sync ręcznie do plan.md)

curl -X POST https://jb69vusexb.execute-api.eu-central-1.amazonaws.com/agent/action -H "Content-Type: application/json" -d '{"action":"propose","type":"task","title":"Follow-up X","content":"Karpacz prosi ofertę 2027","priority":"high"}'

Type: task / decision / crm_note / fact / context. Zapis idzie do S3 jako markdown.

🤖 8 LAMBD — co która robi

PUSHmail-notify-receiver

Odbiera POST z Gmail Pub/Sub przez API Gateway, parsuje, wpycha do SQS. To jest "drzwi wejściowe" pipeline'u.

SQSmail-processor

Klasyfikuje maila używając RULES R0-R6 (deterministic, conf 1.0) + Bedrock Haiku 4.5 jako AI fallback (gdy rule conf < 0.8). Auto-archive jeśli INFO/NEWSLETTER/TRANSACTIONAL i conf ≥ 0.9.

MANUALmail-drafter (wywołujesz Ty z message_id)

Generuje draft odpowiedzi w Twoim stylu. Czyta z S3 dynamicznie: mail_context_updates.md + ghost.md + profil.md + oferta.md + 15 realnych przykładów Twoich wysłanych maili (sent-samples.json). Bedrock Sonnet 4.6. Post-process: replace em-dash, "Tatusiu"→"Tatuś", "bandów"→"band" itp. (lista WORD_FIXES w kodzie).

APImail-agent-api (/agent/inbox + /agent/action)

5 akcji: send / reject / archive / amend (re-draft z hintem) / propose (zapisz task/decyzję/fakt do S3). Pełny "control panel" Twoich decyzji.

CRONmail-gmail-watch-renew (codziennie 06:00 UTC)

Odnawia Gmail watch dla 3 skrzynek (TTL 7 dni). Bez tego push pada za tydzień. Self-healing.

CRONmail-historical-miner (sobota 07:00 UTC)

Sweep ostatnich 7 dni biuro.gamak — klasyfikuje stare maile (idempotent), wyciąga kontakty do mail-contacts. Można odpalić manual dla full sweep 12 mies wstecz (~52 invocations × 30 sek).

CRONmail-extraction-engine (codziennie 09:00 UTC)

Dla LEAD/KLIENT z ostatnich 50 — wyciąga z body strukturyzowane dane (phone, NIP, role, company) + facts list (np. "Karpacz prosi ofertę na 2027"). Update mail-contacts + facts do S3 (Daniel ręcznie kopiuje wartościowe do mail_context_updates.md).

CRONmail-feedback-analyzer (niedziela 20:00 UTC)

Skanuje feedback (Twoje akcje send/reject/amend) z ostatnich 7 dni → raport JSON do S3 + SNS notify. Pokazuje wzorce: które kategorie najczęściej rejectujesz, jakie hint'y dajesz w amend (kandydaci na nowe reguły).

❓ FAQ

Curl mi nie działa — gdzie wkleić?

Curl to komenda terminala, nie URL do przeglądarki.

  1. Naciśnij Win i wpisz "powershell" (lub "git bash" jeśli masz)
  2. Otwórz terminal
  3. Wklej curl, naciśnij Enter
  4. Wynik to JSON w terminalu — czytaj go tam

PowerShell na nowych Windows ma curl built-in. Git Bash też. Zwykły CMD — nie ma curl, używaj jednego z powyższych.

Drafter wysłał coś czego bym nie wysłał. Co zrobić?

Nigdy NIE wysyłasz auto. Twoja akcja to ZAWSZE manual send. Jeśli draft jest słaby:

  • action=reject — wyrzuć draft, mail oryginalny zostaje w INBOX, sam odpiszesz ręcznie
  • action=amend, hint="..." — Drafter generuje nowy draft z Twoim hintem
  • Jeśli pattern się powtarza (np. "zawsze pisz do JST formalnie") → dodaj regułę do mail_context_updates.md
Skąd Drafter wie że Wiesław to mój tata?

Z pliku gamak/dane/mail_context_updates.md (zsynczowany do S3 jako kontekst). Reguła:

wklimczak.sportmanager@gmail.com → Wiesław Klimczak (TATA Daniela)
Pierwsza linia: "Tatuś," (NIE "Tato", NIE "Wiesławie", NIE "Tatusiu")
Tone: ciepły rodzinny ale często biznesowy
Często żartobliwie. Bez signoff, kończ "Daniel" lub naturalnie
Jak wyłączyć auto-archive (znika za dużo dobrych maili)?

W terminalu:

aws lambda update-function-configuration --region eu-central-1 --function-name mail-processor --environment "Variables={...,AUTONOMOUS_MODE=off,...}"

Lub podnieś próg: zmień AUTO_ARCHIVE_THRESHOLD z 0.9 na 0.95 (więcej będzie czekało na ręczną decyzję).

Co się stanie jak Pub/Sub umrze?

SQS DLQ łapie błędy (email-inbox-dlq). Alarm mail-dlq-not-empty powiadomi Cię na email. Plus zawsze możesz manual: aws lambda invoke --function-name mail-processor --payload '{"count":10}' — pobierze ostatnie 10 maili ręcznie.

Gdzie pełna treść moich maili?

Treść jest TYLKO w Twoim Gmail. Lambdy pobierają na żądanie (gdy potrzeba), nie zapisują pełnego body do DDB. W mail-emails jest: from, subject, snippet (200 chars), klasyfikacja, status. Pełen body zostaje w skrzynce Gmail.
